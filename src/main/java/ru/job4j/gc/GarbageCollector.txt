User user1 = new User();

в 64-х разрядной JVM HotSpot будут занимать
user1 - 24 байт:
12 байт (заголовок) + 4 байт (int) + 4 байт (ссылка) + 4 байта (выравнивание);
2 идентичных  объекта  =  48 байт и т.д.

Пустой объект занимает 16 байт в 64-х разрядной JVM HotSpot
(12 байт заголовок объекта, 4 байта для выравнивание под два машинных слова).

Объект new User(100, "U100") - занимает в памяти 72 байт
12 байт (заголовок)  + 4 байта (выравнивание)
+ 8 байт поле int (4 байт (int) + 4 байт (ссылка))
+ 48 байт поле String


При ограничении хипа -Xmx6m -Xms2m и создании в цикле объектов User без ссылки
примерно на 3700-ой итерации происходит вызов GC,
обусловленный заполнением хипа молодого поколения и необходимостью  его очищать.
